# Regular Expressions
"Some people, when confronted with a problem think 'I know, I'll use regular expressions.' Now they have two problems." - Jamie Zawinski

## ● What is it?
```javascript 
str = "this_isSpinal Tap";
str.split(/\s|_|(?=[A-Z])/).join("-");
```
>* Regex or RegExp forms a small, separate language for inspecting strings. 
>* It is used to match character combinations.




---

## ● Creating a Regex Pattern
```javascript
var rg = /ab+c/;
```
```javascript
var rg = new RegExp('ab+c');
```



---


## ● Sets of Characters
/abc/ matches this sequence
[abc] matches any one of these
### ╸ Range
[0-9A-Za-z] matches any character in range (Unicode)
### ╸ Repeating
/ab+c/ matches 'abc', 'abbc', and 'abbbbbbc'
/ab\*c/ matches 'abc' or 'ac'
/neighbou?r/ matches 'neighbor' or 'neighbour'

### ╸ Boundaries and pipes
/\b/ matches beginning or end
/^(www)/ matches at beginning
/(.com)$/ matches at end
/a|b|c/




---

## ● Testing a Regex Pattern
#### ╸ test returns Boolean
/ee/.test('Meee');  //true 

#### ╸ match returns Index
str.match(/ee/);  //1

#### ╸ exec returns an array of info
/ee/.exec('Meee');  //['ee', '1']

#### ╸ .replace
str.replace(/[\W_]/gi, '')
#### ╸ .search() not .indexOf()
returns index or -1

## ● Real-world Example

```javascript=
  var phoneTest = "(555) 555-5555";
  var phoneRE = /^\(\d\d\d\) \d\d\d-\d\d\d\d$/; 
  if (phoneTest.match(phoneRE)) {
    console.log(true);  // Valid phone number
  } else {
    console.log(false); //Invalid phone number
  }   
```

## ● Real-world Example - email Validator
```javascript=
var validEmail = "tamer@gmail.com";
var invalidEmail = "tamer.com";
var invalidEmail2 = "tamer";
var re = /\S+@\S+\.\S+/;
console.log(re.test(validEmail)); // true
console.log(re.test(invalidEmail)); // false
console.log(re.test(invalidEmail2)); // false
```

## ● Real-world Example - Date Validator
```javascript=
var dateReg = /^\d{2}[./-]\d{2}[./-]\d{4}$/
console.log("22-03-1981".match(dateReg));//true
console.log("22.03-1981".match(dateReg)); //false
console.log("22.03.1981".match(dateReg)) //true

```

* It is most helpful to inspect string snippets/substrings.
* Keep it simple!
